# 💡 Java 基本类型与引用类型详解笔记

---

## 一、Java 数据类型分类

### ✅ 1. 基本类型（Primitive Types）- 值类型（共8种）

|类型|说明|默认值|
|---|---|---|
|int|整数（32位）|0|
|long|长整数|0L|
|float|单精度浮点数|0.0f|
|double|双精度浮点数|0.0|
|boolean|布尔类型|false|
|char|字符|'\u0000'|
|byte|字节|0|
|short|短整数|0|

📌 特点：

- 存储在栈中
    
- 赋值、传参时传的是值的副本
    
- 改副本不会影响原值
    

---

### ✅ 2. 引用类型（Reference Types）

|类型|举例|
|---|---|
|类 Class|String, Scanner, 自定义类|
|数组 Array|int[], String[]|
|接口 Interface|List, Map|
|枚举 Enum|enum 类型|

📌 特点：

- 存储在堆中，对象变量中保存的是**引用（地址）**
    
- 赋值和传参时，传的是**地址值的副本**
    
- 可通过引用修改对象内容，但不能改变原引用变量的指向
    

---

## 二、赋值行为对比

|操作|基本类型|引用类型|
|---|---|---|
|赋值|复制值|复制地址（引用）|
|改副本是否影响原变量|❌ 不影响|✅ 会影响指向的对象内容|
|是否能改原引用地址|❌ 不能|❌ 不能（引用本身是值传递）|

---

## 三、方法传参：值传递本质

> Java 一切都是“值传递”，包括对象引用本身。

|类型|行为|示例|
|---|---|---|
|基本类型|传值的副本，不影响原始变量|`x = 10 → x = 100` 不影响外部|
|引用类型|传的是地址值的副本，可改对象内容但不能改引用本身|`n.data = 99` 有效；`n = new Node()` 无效|

---

### ✅ 示例

```java
void test(Node n) {
    n.data = 99;           // 改对象内容 ✅
    n = new Node(200);     // 改引用指向 ❌
}
```

```java
Node a = new Node(10);
test(a);
System.out.println(a.data); // 输出 99（不是 200）
```

---

## 四、内存模型理解图

```
Node a = new Node(10);
```

内存布局如下：

```
[栈] a ──► [堆] Node 对象 { data: 10, next: null }
```

方法参数传递时：

- 基本类型：直接复制值
    
- 引用类型：复制引用（地址值）

## 五、面试常见问题整理

|问题|正确答案/思路|
|---|---|
|Java 是值传递还是引用传递？|✅ 值传递|
|方法内改变引用对象属性是否有效？|✅ 有效|
|方法内改变引用指向是否有效？|❌ 无效|
|为什么对象传参能改属性？|因为地址值是复制的，但对象是共享的|
|基本类型传参会改变原变量值吗？|❌ 不会|

## 六、装箱与拆箱（扩展）

Java 提供包装类用于把基本类型转换成对象：

|基本类型|包装类|
|---|---|
|int|Integer|
|double|Double|
|boolean|Boolean|
|char|Character|

```java

```

## 七、口诀记忆

```
基本类型值独立，拷贝传参无牵连；
对象类型传地址，共享内容可更改；
引用变更没影响，原指向仍未变；
方法传参皆值传，牢记本质避混乱！
```

## 🌟 示例 1：基本类型传参（值拷贝）

```java
public class Example1 {
    public static void changeInt(int x) {
        x = 100;
    }

    public static void main(String[] args) {
        int a = 10;
        changeInt(a);
        System.out.println("a = " + a); // 输出：a = 10
    }
}
```

🧠 **解析**：

- `a` 的值传给 `x`，是 **值的副本**
    
- 修改 `x` 不会影响原来的 `a`
    

---

## 🌟 示例 2：引用类型传参（可修改对象内容）

```java
class Node {
    int data;
    Node(int data) {
        this.data = data;
    }
}

public class Example2 {
    public static void modifyNode(Node node) {
        node.data = 999;  // 修改节点内容
    }

    public static void main(String[] args) {
        Node a = new Node(100);
        modifyNode(a);
        System.out.println("a.data = " + a.data); // 输出：a.data = 999
    }
}
```

🧠 **解析**：

- `node` 是 `a` 的引用副本
    
- 通过 `node.data = ...` 实际修改了堆中的对象数据，`a.data` 跟着变
    

---

## 🌟 示例 3：试图改变对象引用（失败）

```java
class Node {
    int data;
    Node(int data) {
        this.data = data;
    }
}

public class Example3 {
    public static void reassign(Node node) {
        node = new Node(888); // 修改了 node 的引用指向
    }

    public static void main(String[] args) {
        Node a = new Node(100);
        reassign(a);
        System.out.println("a.data = " + a.data); // 输出：a.data = 100
    }
}
```

🧠 **解析**：

- 方法中 `node = new Node(...)` 只是让形参引用指向新对象
    
- 不影响原来的变量 `a`，所以 `a.data` 仍然是 100
    

---

## 🌟 示例 4：修改数组（引用类型）

```java
public class Example4 {
    public static void modifyArray(int[] arr) {
        arr[0] = 42; // 修改数组内容
    }

    public static void main(String[] args) {
        int[] nums = {1, 2, 3};
        modifyArray(nums);
        System.out.println(nums[0]); // 输出：42
    }
}
```

🧠 **解析**：

- `int[]` 是引用类型，`arr` 和 `nums` 指向同一个数组
    
- 所以对 `arr[0]` 的修改影响到 `nums`
    

---

## 🌟 示例 5：值传递 vs. 引用传递比较（综合）

```java
public class Example5 {
    public static void test(int x, int[] y) {
        x = 50;
        y[0] = 999;
    }

    public static void main(String[] args) {
        int a = 10;
        int[] b = {1, 2, 3};
        test(a, b);
        System.out.println("a = " + a);      // 输出：a = 10（基本类型不变）
        System.out.println("b[0] = " + b[0]); // 输出：b[0] = 999（引用类型改变）
    }
}
```

🧠 **解析总结**：

- `a` 是基本类型，方法内修改无效
    
- `b` 是引用类型，修改数组元素有效

## 🧠 小结口诀回顾：

```
传 int 是复制值，改了也不会动；
传对象是地址，改属性真生效；
想换指向没用处，值传递记心中。
```
