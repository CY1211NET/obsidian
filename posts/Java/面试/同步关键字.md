# 🟥 一、进程与线程到底是什么？（最简单版本）

## ✔ 进程（Process）是什么？

> **进程 = 运行着的程序本身 + 独立的一块内存空间。**

例如：

- 打开微信 → 一个进程
    
- 打开 Chrome → 一个进程
    
- 打开 IDEA → 一个进程
    

每个进程：

- 有自己独立的 **内存空间**
    
- 进程之间不能直接访问对方的内存（隔离）
    

---

## ✔ 线程（Thread）是什么？

> **线程 = 在进程内部“执行代码的一个最小执行单位”。**

一个进程内部可以有多个线程共同工作。

例如浏览器：

- 1 个线程处理 UI
    
- 1 个线程加载网页
    
- 1 个线程执行 JS
    
- 1 个线程播放视频
    

多个线程共享：

- **堆内存（对象、全局变量）**
    
- **方法区（类信息）**
    

但每个线程都有：

- 自己的栈（方法调用栈）
    
- 自己的程序计数器（PC）
    
- 自己的线程上下文

# 🟦 二、线程之间“看不到彼此修改”的问题（核心）

**因为：每个线程都有 CPU 缓存！**

看这个示例：

```java
public class Demo {
    static boolean running = true;

    public static void main(String[] args) throws Exception {
        new Thread(() -> {
            while (running) {
            }
            System.out.println("Thread end");
        }).start();

        Thread.sleep(1000);
        running = false;
        System.out.println("running 置为 false");
    }
}
```

理论效果：

- 主线程把 `running` 改成 false
    
- 子线程应该退出循环
    

👉 **实际上，有时子线程会永远卡死，不退出！**

为什么？

**因为子线程读取到的是“旧值”，它没看到主线程的修改。**

---

# 🟥 三、为什么看不到？（线程的可见性问题）

原因是：

> 每个线程运行时，数据会被读到 CPU 的 L1/L2 缓存中，  
> 而不会每次都直接去主内存读。

流程类似：

```
主线程       子线程
│            │
│ running = false (写入主内存)
│            │ running = true (读的是自己缓存里的旧值)
│            │ 死循环……
```

子线程读取的是缓存在自己本地 CPU 缓存中的数据，而不是主内存。

---

# 🟦 四、如何让线程看到最新值？（volatile 解决可见性）

加 volatile：

```java
static volatile boolean running = true;
```

volatile 的作用：

1. **保证可见性**
    
    - 每次写入都会刷新到主内存
        
    - 每次读取都会从主内存读取最新值
        
2. **禁止指令重排序**（额外好处）
    

加 volatile 后，子线程就能“看到”主线程改的值。

---

# 🟥 五、线程可见性的真实图解

### ✔ 普通变量流程（可能不可见）

```
主线程写 ---------> 主内存
                     ↑
子线程读 <--------- CPU 缓存（旧值）
```

### ✔ volatile 变量流程（强制可见）

```
主线程写 ----刷新----> 主内存
                       ↑
子线程读 ----读取最新值---│
```

这就是“可见性”的本质。

---

# 🟪 六、那 synchronized 为什么也能保证可见性？

因为 synchronized 具有两大能力：

1. **互斥（只有一个线程能进锁）**
    
2. **可见性（释放锁前必须把缓存刷回主内存，下一个线程拿锁时必须从主内存读）**
    

工作原理：

```
线程 A 获得锁 → 读数据 → 修改数据 → 释放锁（强制写回主内存）
线程 B 获取锁 → 从主内存读取最新数据
```

# 🟩 七、进程 vs 线程：再简单总结

|概念|含义|内存是否共享|
|---|---|---|
|进程|程序运行的整体|❌ 进程间不共享|
|线程|进程中的一个“执行路径”|✔ 同一进程的线程共享堆内存|

---

# 🟦 八、可见性 vs 原子性 vs 有序性

并发编程三大问题：

|术语|含义|volatile 能保证吗？|
|---|---|---|
|可见性|线程看到最新变量值|✔ 保证|
|原子性|操作不可分割|❌ 不保证|
|有序性|禁止重排序|✔ 保证|

如果你同时需要可见性 + 原子性，就要用 `synchronized` 或原子类 `AtomicInteger`。

---

# 🟧 九、一句话让你记住线程可见性

> **一个线程修改变量后，不一定会让其他线程立刻看到，因为 JVM、CPU 都有缓存；  
> volatile 能强制刷新和读取主内存，从而保证可见性。**

---

# 🟥 十、如果你面试被问“可见性”怎么答？（标准回答）

```
线程可见性指一个线程对共享变量的修改，能否被其他线程立即看到。
由于 CPU 有多级缓存，普通变量可能被缓存在线程的工作内存中，
导致其修改无法及时同步到主内存。

volatile 可以保证可见性，因为它会在写操作时强制刷新到主内存，
在读操作时强制从主内存取最新值。
synchronized 也能保证可见性，因为锁的释放会把变量刷新到主内存，
获取锁时会从主内存读最新值。
```
