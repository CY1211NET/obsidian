# Java 接口继承详解

---

## 一、接口是什么？

- **接口（interface）** 是一种特殊的抽象类型，里面只能定义抽象方法（Java 8 之后也可以有默认方法和静态方法）。
    
- 接口是**规范的集合**，用来规定实现类必须完成的方法。
    
- 不能实例化接口，只能被类实现（`implements`）。
    

---

## 二、接口继承是什么？

- 接口之间也可以继承，使用关键字 `extends`。
    
- 一个接口可以继承一个或多个接口（Java 支持多继承接口）。
    
- 子接口继承了父接口的所有抽象方法和常量。
    

---

## 三、接口继承语法示例

```java
interface A {
    void methodA();
}

interface B extends A {
    void methodB();
}

interface C extends A, B {
    void methodC();
}
```

---

## 四、接口继承的特点

- **多继承**：接口支持多继承，一个接口可以继承多个接口，形成“接口继承树”。
    
- **子接口继承所有父接口方法**，实现类实现子接口时必须实现所有继承过来的抽象方法。
    
- 通过接口继承，可以把多个接口的功能合并，形成更复杂的规范。
    

---

## 五、接口继承和类继承区别

|特点|接口继承|类继承|
|---|---|---|
|继承方式|用 `extends`，支持多继承|用 `extends`，单继承|
|内容|抽象方法、默认方法、静态方法|成员变量、方法、构造器等|
|实现类责任|实现所有抽象方法|继承父类功能，可重写方法|
|多继承支持|支持|不支持|

---

## 六、示例：接口继承应用

```java
interface Worker {
    void work();
}

interface Manager extends Worker {
    void manage();
}

class TeamLeader implements Manager {
    @Override
    public void work() {
        System.out.println("团队领导在工作");
    }

    @Override
    public void manage() {
        System.out.println("团队领导在管理");
    }
}

public class Test {
    public static void main(String[] args) {
        TeamLeader leader = new TeamLeader();
        leader.work();
        leader.manage();

        // 接口引用也可以指向实现类
        Worker w = leader;
        w.work();

        Manager m = leader;
        m.manage();
    }
}
```

---

## 七、小结

- 接口继承允许接口之间建立层次结构，增强接口的复用和扩展能力。
    
- 多继承接口是接口灵活的关键特性，方便组合功能。
    
- 实现类实现子接口时，需要实现所有父接口的方法。
