# 🌟 Java 设计模式面试题与答案速记表

---

## 一、基础理解题

### 1️⃣ 什么是设计模式？为什么要用？

**答案：**

- 设计模式是 **面向对象设计中可复用的解决方案**。
    
- 目的是 **提高系统可维护性、可扩展性和可复用性**，避免“重复造轮子”。
    
- 面试思路：强调 **抽象问题、复用解决方案、提高代码质量**。
    

---

### 2️⃣ 设计模式分类？

**答案：**

|类型|作用|常见模式|
|:--|:--|:--|
|创建型|对象创建|单例、工厂方法、抽象工厂、建造者、原型|
|结构型|对象组合|代理、装饰器、适配器、桥接、外观、组合、享元|
|行为型|对象协作|策略、观察者、模板方法、责任链、状态、命令、迭代器、备忘录、中介者、解释器、访问者|

## 二、创建型模式

### 3️⃣ 单例模式的作用与实现？

**答案：**

- **作用**：保证系统中某个类只有一个实例，并提供全局访问。
    
- **实现方式**：懒加载 + 双重校验 + volatile（线程安全）
    

```java
class Singleton {
    private static volatile Singleton instance;
    private Singleton() {}
    public static Singleton getInstance() {
        if(instance==null){
            synchronized(Singleton.class){
                if(instance==null) instance = new Singleton();
            }
        }
        return instance;
    }
}
```

- 面试延伸：
    
    - 为什么 volatile 必须加？
        
        > 防止指令重排，保证对象初始化可见性。
        
    - 饿汉式 vs 懒汉式区别？

### 4️⃣ 工厂模式的作用？

**答案：**

- **工厂方法模式**：将对象创建与使用解耦，符合开闭原则。
    
- **示例**：支付方式（微信支付、支付宝）
    

```java
interface Product { void show(); }
class AProduct implements Product { public void show(){System.out.println("A");} }
class Factory {
    public static Product create(String type){
        if("A".equals(type)) return new AProduct();
        return null;
    }
}
```

---

### 5️⃣ 建造者模式？

**答案：**

- **作用**：构建复杂对象，将构建与表示分离
    
- **应用**：链式调用、复杂对象初始化
    

```java
class Computer {
    private String cpu, ram, disk;
    public Computer setCpu(String cpu){this.cpu=cpu; return this;}
    public Computer setRam(String ram){this.ram=ram; return this;}
    public Computer setDisk(String disk){this.disk=disk; return this;}
}
```

## 三、结构型模式

### 6️⃣ 代理模式？

**答案：**

- **作用**：控制访问、增强方法
    
- **应用**：AOP、RPC、权限控制
    

```java
interface Service { void request(); }
class RealService implements Service { public void request(){System.out.println("真实请求");} }
class ProxyService implements Service {
    private RealService target = new RealService();
    public void request(){
        System.out.println("权限校验");
        target.request();
        System.out.println("日志记录");
    }
}
```

---

### 7️⃣ 装饰器模式？

**答案：**

- **作用**：动态给对象添加功能
    
- **区别于继承**：继承是静态增加，装饰器是动态增加
    
- **示例**：I/O 流
    

```java
interface Coffee { double getCost(); String getDescription(); }
class SimpleCoffee implements Coffee { 
    public double getCost(){ return 10; }
    public String getDescription(){ return "普通咖啡"; }
}
class MilkDecorator implements Coffee {
    private Coffee coffee;
    public MilkDecorator(Coffee coffee){this.coffee=coffee;}
    public double getCost(){return coffee.getCost()+3;}
    public String getDescription(){return coffee.getDescription()+" + 牛奶";}
}
```

---

### 8️⃣ 适配器模式？

**答案：**

- **作用**：接口转换，解决不兼容问题
    
- **应用**：USB 转 Type-C
    

```java
interface Target { void request(); }
class Adaptee { void specificRequest(){System.out.println("老接口");} }
class Adapter implements Target {
    private Adaptee adaptee;
    public Adapter(Adaptee adaptee){this.adaptee=adaptee;}
    public void request(){adaptee.specificRequest();}
}
```

## 四、行为型模式

### 9️⃣ 策略模式？

**答案：**

- **作用**：封装算法，可互换
    
- **应用**：支付方式选择、排序策略
    

```java
interface Strategy { int operate(int a,int b); }
class Add implements Strategy { public int operate(int a,int b){return a+b;} }
class Context {
    private Strategy strategy;
    public Context(Strategy s){this.strategy=s;}
    public int execute(int a,int b){ return strategy.operate(a,b);}
}
```

---

### 10️⃣ 观察者模式？

**答案：**

- **作用**：一对多通知机制
    
- **应用**：事件监听、消息推送
    

```java
interface Observer { void update(String msg); }
class User implements Observer {
    private String name;
    public User(String n){this.name=n;}
    public void update(String msg){System.out.println(name+" 收到:"+msg);}
}
class Wechat {
    private List<Observer> list = new ArrayList<>();
    public void add(Observer o){list.add(o);}
    public void notifyAll(String msg){for(Observer o:list) o.update(msg);}
}
```

---

### 11️⃣ 模板方法模式？

**答案：**

- **作用**：固定算法框架，子类实现具体步骤
    
- **应用**：JdbcTemplate、Servlet
    

```java
abstract class Game {
    abstract void start();
    abstract void end();
    public final void play(){
        start();
        System.out.println("游戏进行中...");
        end();
    }
}
class LOL extends Game {
    void start(){System.out.println("LOL开始");}
    void end(){System.out.println("LOL结束");}
}
```

### 12️⃣ 责任链模式？

**答案：**

- **作用**：请求沿链传递，直到处理
    
- **应用**：Spring FilterChain、Netty ChannelPipeline

## 五、设计原则（面试常问）

|原则|面试答法|
|:--|:--|
|单一职责|一个类只做一件事|
|开闭原则|对扩展开放，对修改关闭|
|里氏替换|子类可替换父类对象|
|依赖倒置|依赖抽象而非实现|
|接口隔离|不强迫实现不需要的接口|
|迪米特法则|最少知道原则，降低耦合|
