## 一、方法重写（Override）

### 定义

方法重写是子类对父类中的方法重新实现。子类继承父类后，可以根据需要改写父类中的方法，以实现不同的功能。

### 关键点

- **方法名、参数列表、返回类型**必须与父类被重写的方法完全一致。
    
- 访问修饰符不能比父类更严格（比如父类是`public`，子类不能写成`private`）。
    
- 只能重写实例方法，不能重写`static`方法（静态方法不能被重写，只能被隐藏）。
    
- 子类重写的方法可以调用父类的被重写方法（`super.method()`）。
    
- 重写是**运行时多态**的基础，方法调用由运行时对象类型决定。
    

### 例子

```java
class Animal {
    void sound() {
        System.out.println("动物发出声音");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("狗叫");
    }
}
```

调用时：

```java
Animal a = new Dog();
a.sound();  // 输出：狗叫
```

### 前瞻视角

- 重写使得多态得以实现，便于扩展和维护。
    
- 结合接口和抽象类，重写是实现设计模式如策略模式、模板方法的重要技术。
    
- 在Java 8+中，重写也可以配合**默认方法（default methods）**使用，增强接口灵活性。
    
- 在现代项目中，重写和多态对于实现可插拔架构、动态行为切换非常关键。

## 二、方法重载（Overload）

### 定义

方法重载是同一个类中，方法名相同但参数列表不同（参数个数或类型不同），实现不同功能的方法。

### 关键点

- 方法名必须相同。
    
- 参数列表必须不同（参数类型、个数、顺序）。
    
- 与返回值无关，返回类型不同不能构成重载。
    
- 重载是在**编译时确定**调用哪个方法，属于静态多态。
    
- 重载提高程序的可读性和灵活性。
    

### 例子

```java
class MathUtils {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

调用时：

```java
MathUtils mu = new MathUtils();
mu.add(1, 2);       // 调用第一个
mu.add(1.0, 2.0);   // 调用第二个
mu.add(1, 2, 3);    // 调用第三个
```

### 前瞻视角

- 重载让API设计更简洁，减少了方法名爆炸，尤其在Java库和框架中很常见。
    
- 与可变参数（varargs）、泛型结合，能设计出更灵活的接口。
    
- 在函数式编程趋势下，重载可以与Lambda表达式结合，实现更灵活的函数调用。
    
- 随着项目复杂度提升，重载可以帮助实现多场景的调用契约，提高代码复用。

## 三、重写 vs 重载 总结对比表

|特性|方法重写（Override）|方法重载（Overload）|
|---|---|---|
|定义|子类重新定义父类方法|同类中同名不同参数方法|
|发生位置|父类与子类之间|同一类内部|
|参数列表|必须完全相同|必须不同|
|返回类型|必须相同或协变返回类型|不影响重载|
|访问权限|不能更严格|不影响重载|
|多态类型|运行时多态|编译时多态|
|作用|实现多态，改变父类行为|提供多种调用方式，提高灵活性|

---

## 四、应用建议（Forward-looking）

- **设计接口和类库时**，合理使用重载提供简洁且多功能的API入口。
    
- **子类实现时**，重写父类或接口方法，实现业务特化和多态行为。
    
- 利用Java 8及以上的新特性（如默认方法、函数式接口）结合重写，实现更灵活的扩展。
    
- 关注代码规范，重写方法使用`@Override`注解，减少错误。
    
- 未来的动态语言特性和字节码操作技术（如字节码增强、代理）中，重写机制依然是核心。
